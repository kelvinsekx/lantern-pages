Email has become an ubiquitous part of modern application. As a primary communication channel, nearly every user interaction, from password resets to transaction receipts, triggers an automated email message.

In this module, we'll explore internet mails (email), its lifecycle, and the protocols that carries them from the senders client to the receivers inbox and choices you face as a developer to implement one that scale in Node and NestJS.

## The 6 pillars of email delivery

The email ecosystem relies on six critical components I call pillars. Much like a physical postal service, there is a lot happening before your letter or item reaches the intended recipient.

1. **Mail User Agent (MUA)**,commonly known as the email client is the interface (Google Gmail, Microsoft Outlook, Mozilla Thunderbird) where the sender composes the message before being sent to a sender's email server.
2. Mail Submission Agent (MSA) is a crucial component of email delivery. The software does three (3) things: **accepts the mail from the MUA**, **validate & checks for errors** before **sending mail to the MTA**.
3. Mail Transfer Agent (MTA) is responsible for the actual transfer of email messages between servers. It routes the email across the internet to the recipient's server.
4. Receiving MTA: The destination server that accepts the incoming message.
5. Mail Delivery Agent (MDA): The "local carrier" that places the message into the recipient's specific mailbox.
6. IMAP/POP3 Protocols: The tools the recipient uses to retrieve the mail from the mailbox.

These components can be further categorized into two

- Email Infrastructure: user agents, servers, mailboxes
- SMTP: simple mail transfer protocol

We won't go into details with MSAs, MTAs. However, this article will be incomplete without discussing in details some of the protocols used for sending emails.

## Understanding SMTP

SMTP (Simple Mail Transfer Protocol) is a one way protocol used for sending email only (not responsible for receiving email) that sits on top of TCP. Think of it as a one way post office for your emails. They can send them but not receive them. It's important to add that unlike other TCP meta-protocols like HTTP that pulls data from a server, SMTP pushes data to a server or between servers.

To complete the loop, SMTP hands off the message to the MDA, where protocols like IMAP or POP3 allow the end-user to access these messages in their inboxes.

While SMTP remains the backbone of email, many modern developers are shifting toward HTTP-based APIs (like SendGrid or Postmark). These offer better security (hiding your server's IP), easier credential management, and built-in analytics that raw SMTP lacks.
